# FormFx への貢献 (Contributing)

FormFxへの貢献を検討していただき、ありがとうございます！

## 開発セットアップ

1.  **リポジトリをクローンする**
2.  **依存関係をインストールする**:
    ```bash
    npm install
    ```
3.  **ウォッチモードでビルドする**:
    ```bash
    npm run dev
    ```
4.  **テストを実行する**:
    ```bash
    npm test
    ```

## プロジェクト構造

- `src/core`: メインの `FormFx` クラスとエントリポイント。
- `src/internal`: プライベートな実装詳細（トークナイザー、パーサー、評価器）。**ここのアイテムを `src/index.ts` でエクスポートしないでください。**
- `src/plugins`: リピーターなどの公式プラグイン。
- `src/editor`: ルールエディタの実装（メインバンドルから分離されています）。
- `src/public-types.ts`: 厳密に定義された公開型。

## プルリクエストのガイドライン

- すべてのテストがパスすることを確認してください。
- 新機能やバグ修正にはテストを追加してください。
- セマンティック バージョニング (SemVer) に従ってください。v1.x では破壊的変更を行わないでください。
- `CHANGELOG.md` に変更内容を追記してください。

## テスト

ユニットテストおよび統合テストには [Vitest](https://vitest.dev/) を使用しています。
E2Eテストには [Playwright](https://playwright.dev/) を使用しています。

```bash
# ユニットテスト/統合テストを実行
npm test

# E2Eテストを実行 (Playwrightのインストールが必要)
npm run test:e2e
```
